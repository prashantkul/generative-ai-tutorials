Transformers

Attention is all you need — how letting every token talk to every other token changed everything.

By Prashant Kulkarni · with help from Claude Opus CC BY-NC 4.0
1 Motivation

Why attention?

Before transformers, recurrent neural networks processed sequences one token at a time, threading a hidden state forward through the chain: $h_t = f(h_{t-1}, x_t)$. This design created three problems that attention solves simultaneously.

  • Sequential computation prevents parallelism. You cannot compute $h_{100}$ without first computing $h_1$ through $h_{99}$. On GPUs with thousands of parallel cores, this serialization is catastrophic.
  • Vanishing gradients over long chains. Backpropagation through $T$ timesteps involves a product of $T$ Jacobians. When each has spectral norm slightly less than 1, the product vanishes exponentially.
  • Information bottleneck. The entire history of the sequence must be compressed into one fixed-dimensional vector $h_t$. By token 500, information about token 1 has been repeatedly overwritten.

The key insight: let every token look at every other token directly, without routing information through a chain of intermediate states. Computation becomes parallel. Gradient paths become short (one step, not $T$). There is no bottleneck. The cost is quadratic — $O(n^2)$ for a sequence of length $n$ — but this is a price modern hardware pays gladly.

2 Core mechanism

Scaled dot-product attention

Attention is fundamentally a soft dictionary lookup. We have three ingredients:

  • Query (Q) — "what am I looking for?" Each token produces a query vector that encodes what information it needs from other tokens.
  • Key (K) — "what do I contain?" Each token produces a key vector that advertises what information it offers.
  • Value (V) — "what information do I provide?" Each token produces a value vector — the actual content to be gathered.

These are computed by projecting the input through learned weight matrices:

$$Q = XW_Q, \qquad K = XW_K, \qquad V = XW_V$$

The attention computation itself:

$$\text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{QK^T}{\sqrt{d_k}}\right) V$$
Why divide by √d_k?

When $d_k$ is large, the dot products in $QK^T$ grow large in magnitude (variance scales with $d_k$). Large values push softmax into regions where gradients are nearly zero — the "saturation" problem from §1 of Foundations. Dividing by $\sqrt{d_k}$ rescales the variance back to approximately 1, keeping the softmax well-conditioned. This is the same variance-preserving trick that appears in weight initialization and diffusion noise schedules.

3 The attention matrix

What it looks like, and causal masking

The matrix $\text{softmax}(QK^T / \sqrt{d_k})$ is an $n \times n$ attention matrix. Entry $(i, j)$ tells you how much token $i$ attends to token $j$. Each row sums to 1 (it's a probability distribution over which tokens to look at).

For a language model, we need causal masking: token $i$ should only attend to tokens at positions $\leq i$ (no peeking at the future). We do this by setting the upper-triangle entries to $-\infty$ before the softmax, which drives them to zero:

4 Multi-head attention

Multiple views, in parallel

A single attention head computes one set of weights — one "view" of which tokens are relevant. But relevance is multifaceted. In "The cat sat on the mat because it was tired," the word "it" needs to attend to "cat" (coreference), "sat" (action), and "tired" (state) — different relationships that a single distribution cannot capture simultaneously.

Multi-head attention runs $h$ independent attention operations in parallel, each with its own $W_Q^{(i)}, W_K^{(i)}, W_V^{(i)}$ projection, then concatenates and projects the results:

$$\text{MultiHead}(X) = \text{Concat}(\text{head}_1, \ldots, \text{head}_h)\, W_O$$

Each head operates in a $d_k = d_\text{model}/h$ dimensional subspace. With $h=12$ heads and $d_\text{model}=768$ (GPT-2 small), each head has dimension 64. The total parameter cost is $4d_\text{model}^2$ — the same as a single-head version, but with far more representational flexibility.

What heads learn (empirically)

Studies of trained transformers show that heads specialize: some learn positional patterns (attend to the previous token), some learn syntactic relationships (subject-verb agreement), some learn semantic similarity, and some are near-degenerate and could be pruned. This specialization emerges from training — it is not engineered.

5 Position

Positional encoding

Attention treats its input as a set — it is permutation-equivariant. "The dog bit the man" and "The man bit the dog" produce the same attention scores (just shuffled). But order matters enormously in language. Positional encodings inject position information.

Three approaches, in historical order:

  • Sinusoidal (original transformer, 2017): fixed sine/cosine functions at different frequencies: $PE_{(\text{pos}, 2i)} = \sin(\text{pos}/10000^{2i/d})$. Zero parameters. Can theoretically extrapolate to longer sequences. Elegant: relative position is recoverable via a linear transformation.
  • Learned (BERT, GPT-2): a trainable matrix $P \in \mathbb{R}^{n_\text{max} \times d}$. More flexible but cannot extrapolate beyond $n_\text{max}$.
  • RoPE (LLaMA, Mistral, modern LLMs): rotates Q and K vectors in 2D subspaces by position-dependent angles. The dot product $\tilde{q}_m^T \tilde{k}_n$ depends only on relative position $m - n$. Best extrapolation; dominant in practice.
6 The block

The transformer block — the repeating unit

A single transformer block combines multi-head attention with a feed-forward network (MLP), connected by residual connections and layer normalization. The standard "pre-norm" layout:

$$h' = \text{Attention}(\text{LN}(h)) + h$$
$$h'' = \text{MLP}(\text{LN}(h')) + h'$$

A transformer is just $N$ of these blocks stacked. GPT-2 small has 12; GPT-3 has 96; LLaMA 70B has 80. The input enters the first block as token embeddings plus positional encodings; the output of the last block feeds into the prediction head.

The MLP inside each block is typically a two-layer network that expands the dimension by 4× then compresses back: $d_\text{model} \to 4 \times d_\text{model} \to d_\text{model}$, with GELU activation between. This is where most of the parameters live — about two-thirds of a transformer's parameters are in the MLP layers (recall MoE, Chapter 10, which replaces these with specialist experts).

7 Architectures

Encoder, decoder, or both

Encoder-only

Bidirectional attention (every token sees every other). Good for understanding: classification, NER, similarity. BERT is the classic example.

Decoder-only

Causal (left-to-right) attention. Good for generation: language modeling, code, chat. GPT, LLaMA, Mistral — nearly all modern LLMs.

There is also the encoder-decoder architecture (T5, BART, the original transformer for machine translation), which uses bidirectional attention in the encoder and causal + cross-attention in the decoder. Covered in depth in Chapter 08.

"Today, decoder-only is the default for language models. If someone says 'transformer' without qualification, they almost certainly mean a causal decoder."

9 Training

Next-token prediction — the only objective

The training objective for a causal language model is shockingly simple: given a sequence of tokens $x_1, x_2, \ldots, x_n$, maximize the probability of each token given all preceding tokens:

$$\mathcal{L} = -\frac{1}{n}\sum_{t=1}^{n} \log P_\theta(x_t \mid x_1, \ldots, x_{t-1})$$

This is cross-entropy loss — the same loss from Chapter 01 §5. The model sees a batch of text, predicts the next token at every position (in parallel, thanks to the causal mask), and receives a gradient for every prediction. No labels needed beyond the text itself — the next token is the label. This is self-supervised learning: the data provides its own supervision.

The unreasonable effectiveness of next-token prediction

To predict the next token well, the model must learn syntax, semantics, reasoning, world knowledge, common sense, and style — all compressed into a single objective. This is why scaling works: more parameters and more data directly translate to better next-token prediction, which translates to better performance on essentially every downstream task.

10 Field notes

Training a transformer on sequence reversal

Source

Experiments from the Module 02 notebook in the learn-generative-ai repo.

Why sequence reversal?

It's a task that requires attention to solve well. The model must learn that the token in output position 1 should attend to input position $n$, position 2 to position $n-1$, and so on. A model that merely copies or predicts based on local context will fail. When it succeeds, we can verify it learned the right attention pattern by visualizing the attention maps.

Attention maps reveal the learned algorithm

After training, extracting the attention weights from the output positions reveals a strikingly clean anti-diagonal pattern: output token 1 attends almost exclusively to input token $n$, output token 2 to $n-1$, and so on. The model has literally learned to "look at the right place" — the attention mechanism is implementing a reversal algorithm, and we can see it directly in the attention matrix.

Head specialization

The notebook also measures attention entropy per head. Some heads have low entropy (sharp, focused attention — looking at exactly one position), while others have high entropy (diffuse attention — gathering context broadly). The reversal-specialized heads have very low entropy — they have learned to attend to exactly one input position per output position. Other heads maintain higher entropy, presumably handling residual contextual tasks.

Temperature sampling

At inference time, varying the temperature of the output softmax changes the model's behavior: at $\tau = 0.1$, the model always picks the highest-probability token (essentially greedy decoding) and gets the reversal right. At $\tau = 2.0$, sampling introduces randomness and the model occasionally makes mistakes — even though it "knows" the right answer, the soft distribution allows incorrect tokens to be sampled.

11 Playground

Interactive: Transformer Explainer

The best way to build intuition for how a transformer works is to watch one run. The Transformer Explainer, built by Cho et al. at Georgia Tech's Polo Club, runs GPT-2 (small, 124M parameters) live in your browser. You can type your own prompt, watch tokens flow through the embedding layer, observe attention patterns across all 12 heads, see the MLP expansion and compression, and experiment with temperature, top-k, and top-p sampling.

Try these experiments in the tool:

  1. Type "The capital of France is" and generate. Look at the attention pattern for "is" — which earlier tokens does it attend to most?
  2. Set temperature to 0.1 (near-deterministic) and generate several tokens. Then set it to 2.0 (high randomness) and regenerate. Compare the diversity of outputs.
  3. Expand the attention view and find a head that shows a strong "attend to previous token" pattern (a diagonal stripe in the attention matrix).
  4. Watch the final softmax distribution. How peaked is it? Does the top token have 90% probability, or is it more spread out?

12 Closing

Takeaways & exercises

The six things to remember

  1. Attention is a soft dictionary lookup: Q asks the question, K advertises the content, V provides the answer, softmax(QKᵀ/√d_k) is the matching score.
  2. The √d_k scaling prevents softmax saturation — the same variance-preserving trick that appears in weight init and diffusion noise schedules.
  3. Multi-head attention gives multiple views at no extra cost — the total parameter count is the same as single-head, but the representational capacity is far higher.
  4. Causal masking makes generation autoregressive — each token can only see tokens before it. This is enforced by setting future positions to $-\infty$ before softmax.
  5. The training objective is just next-token prediction — cross-entropy loss at every position, self-supervised, embarrassingly parallel.
  6. Position must be injected explicitly — attention is a set operation. Sinusoidal, learned, and RoPE are three approaches; RoPE dominates modern LLMs.
Exercises

1. If $Q, K \in \mathbb{R}^{n \times d_k}$, what is the shape of $QK^T$? What does entry $(i,j)$ represent?

2. GPT-2 small has $d_\text{model} = 768$ and $h = 12$ heads. What is $d_k$ per head? How many parameters are in the $W_Q, W_K, W_V, W_O$ matrices combined?

3. Open the Transformer Explainer above. Type "She went to the" and generate. Which token does "the" attend to most strongly in head 0? Does this match your intuition about what "the" needs to predict the next word?

4. Open the Module 02 notebook. Train the transformer on sequence reversal. Visualize the attention maps for the output positions. Do you see the anti-diagonal pattern? Which head(s) learn it most cleanly?

5. Why does the MLP expand by 4× then compress back? What would happen if you removed the MLP entirely and only kept the attention layers? (Hint: think about what attention can and cannot do to individual token representations.)
